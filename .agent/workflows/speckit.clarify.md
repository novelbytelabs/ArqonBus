---
description: Clarify requirements in a specification. Usage: /speckit.clarify
---

1. Analyze the current specification for `[NEEDS CLARIFICATION]` markers or ambiguity.
2. If markers exist:
   - Present specific questions to the user.
   - Update the spec with the user's answers.
3. If no markers exist but ambiguity is found:
   - Propose clarifications to the user.
4. Update the Spec file with the resolved information.
